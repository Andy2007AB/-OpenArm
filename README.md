# -OpenArm
机械臂开源，STM32,板载W25Q64

## 1. 项目概述

本项目是一个基于STM32F103微控制器的机械臂控制系统，使用PS2手柄作为输入设备，实现对机械臂的实时控制。项目包含完整的硬件控制逻辑、逆运动学算法、传感器检测和多串口通信功能。

## 2. 主要功能

### 2.1 核心功能

- **PS2手柄控制**：通过PS2手柄的16个按键实现对机械臂的控制
- **逆运动学分析**：实现机械臂的空间定位和姿态计算
- **舵机控制**：精确控制6个舵机的运动位置和速度
- **传感器检测**：通过外部传感器实现环境感知
- **多串口通信**：支持3个串口的命令接收和发送
- **动作组执行**：支持实时动作组的执行

### 2.2 工作流程

1. 系统初始化：配置GPIO、串口、PS2手柄、传感器等
2. 主循环：
   - 读取PS2手柄数据
   - 处理手柄按键命令
   - 接收并解析串口命令
   - 检测传感器信号
   - 执行相应的机械臂动作

## 5. 机械臂功能大全

### 5.1 舵机控制

- 支持6个舵机的独立控制
- 可设置舵机目标位置和运动时间
- 支持舵机状态监控

### 5.2 逆运动学

- 基于4自由度机械臂的逆运动学算法
- 支持空间坐标到关节角度的转换
- 可配置机械臂各关节长度参数

### 5.3 动作组

- 支持实时动作组的执行
- 支持从存储芯片中读取并执行动作组
- 可通过PS2手柄或串口命令触发动作组
- 实时动作组命令格式：`{#000P1500T1000!#001P1500T1000!}`
- 存储动作组命令格式0：`<GXXXX#000P1500T1000!#001P1500T1000!>`

### 5.4 传感器集成

- 支持外部传感器信号检测
- 可根据传感器信号触发相应动作

### 5.5 多模式控制

- 普通模式：直接控制舵机位置
- AI模式：执行预设的舵机动作序列

## 6. PS2手柄详解

### 6.1 硬件连接

| PS2手柄引脚 | 连接到STM32引脚 |
|------------|----------------|
| CLK | PA12 |
| ATT | PA13 |
| CMD | PA14 |
| DAT | PA15 |
| VCC | 3.3V |
| GND | GND |

### 6.2 通信协议

- 使用软件模拟SPI通信协议
- 主从模式：STM32为主机，PS2手柄为从机
- 通信频率：约500kHz
- 实现方式：通过GPIO模拟SPI时序，手动控制CLK、CMD、ATT引脚状态

## 7. 命令集原理详解

### 7.1 命令格式

#### 7.1.1 系统命令

- 格式：`$命令名称!`
- 示例：
  - `$MODE!`：切换控制模式
  - `$DJR!`：舵机复位
  - `$DST!`：停止所有舵机
  - `$DST:X!`：停止指定舵机（X为舵机编号）
  - `$DGT:X-Y,Z!`：执行动作组（X-Y为动作组范围，Z为执行次数）

#### 7.1.2 舵机控制命令

- 格式：`#舵机索引P目标位置T运动时间!`
- 示例：
  - `#005P0600T2000!`：舵机5移动到600位置，用时2000ms
  - `#000P2400T2000!`：舵机0移动到2400位置，用时2000ms

#### 7.1.3 动作组下载命令

- 保存普通动作组：`<GXXXX#000P1500T1000!#001P1500T1000!>`（XXXX为动作组编号）
- 设置开机动作组：`<$DGT:X-Y,Z!>`（X-Y为动作组范围，Z为执行次数）
- 清除开机动作组：`<$!>`

### 7.2 命令处理流程

1. 接收命令：通过PS2手柄或串口接收命令
2. 解析命令：根据命令格式解析命令类型和参数
3. 执行命令：
   - 系统命令：执行相应的系统功能
   - 舵机控制命令：控制舵机运动
   - 动作组命令：执行预定义动作组
4. 反馈结果：通过串口发送执行结果

### 7.3 命令执行机制

- **PS2手柄命令**：通过手柄按键触发，转换为相应的命令格式
- **串口命令**：直接接收并解析串口数据
- **命令优先级**：PS2手柄命令优先于串口命令

### 7.4 动作组指令处理流程

1. 接收实时动作组命令（如 `{#000P1500T1000!#001P1500T1000!}`）
2. 解析动作组命令，提取每个舵机的控制指令
3. 依次通过串口发送每个舵机的控制指令到舵机总线
4. 舵机执行相应的动作

### 7.5 动作组执行机制

- **动作组存储**：支持将动作组存储到W25Q64存储芯片中
- **解析过程**：
  - 实时动作组：通过 `parse_action` 函数解析实时接收到的动作组命令
  - 存储动作组：通过 `do_group_once` 函数从存储芯片中读取动作组数据，然后解析执行
- **执行过程**：
  - 解析动作组命令为单个舵机控制指令
  - 依次发送到舵机总线
  - 等待动作执行完成
- **控制方式**：动作组命令通过串口3发送到舵机总线
- **开机动作**：支持设置开机动作组，系统启动时自动执行

### 7.6 命令示例

| 命令 | 功能 | 执行效果 |
|------|------|----------|
| `$MODE!` | 切换模式 | 在普通模式和AI模式之间切换 |
| `$DJR!` | 舵机复位 | 所有舵机回到初始位置(1500) |
| `#001P1800T1000!` | 舵机1控制 | 舵机1移动到1800位置，用时1000ms |
| `{#000P1500T1000!#001P1500T1000!}` | 执行实时动作组 | 同时控制舵机0和舵机1移动到1500位置，用时1000ms |

## 8. 系统架构

### 8.1 硬件架构

- **主控芯片**：STM32F103C8T6
- **输入设备**：PS2手柄
- **输出设备**：6个舵机
- **传感器**：外部数字传感器
- **通信接口**：3个串口
  - **USART1**：与上位机通讯
  - **USART2**：预留
  - **USART3**：发送指令到舵机总线控制器

### 8.2 软件架构

- **底层驱动**：GPIO、串口、SPI等硬件驱动
- **中间层**：PS2手柄处理、舵机控制、传感器处理
- **应用层**：命令解析、动作执行、模式管理

### 8.3 数据流

1. 输入数据：PS2手柄数据、串口命令、传感器信号
2. 处理数据：命令解析、逆运动学计算、动作组执行
3. 输出数据：舵机控制信号、串口反馈、蜂鸣器提示

## 9. 技术特点

- **实时控制**：采用中断驱动的串口接收和PS2手柄数据读取
- **精确控制**：支持舵机位置的精确控制和运动时间的设置
- **灵活扩展**：预留了多个GPIO引脚和串口接口
- **多模式操作**：支持普通模式和AI模式的切换
- **模块化设计**：各个功能模块独立封装，便于维护和扩展

## 11. 总结

本项目是一个功能完整的机械臂控制系统，通过PS2手柄实现了对机械臂的直观控制。系统集成了逆运动学算法、舵机控制、传感器检测和多串口通信等功能，具有实时性好、控制精确、扩展性强等特点。
